# Typical Repository Structure

Most starters in this collection follow a similar structure for consistency:

```
repo-name/
├── contracts/          # Smart contracts (Solidity/Move)
├── scripts/            # Deployment scripts
├── test/               # Unit & integration tests
├── frontend/           # React/Next.js examples (optional)
├── INTEGRATION_GUIDE.md
├── README.md
└── .github/workflows/  # CI (on many repos)
```

This structure makes it easy to navigate between different starters.
